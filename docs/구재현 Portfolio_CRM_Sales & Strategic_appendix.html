전략적 Rationale

행동별 가중치는 '성과 전환 확률'에 비례하여 설계되었습니다. 예를 들어, 단순 Visit(2.0)보다 PT(3.5)나 Demo(3.0)가 실제 매출로 이어질 가능성이 1.5배 이상 높다는 현장 데이터(Field Intel)를 반영한 결과입니다.

품질(Quality) 검증 프로세스

모든 입력 데이터는 Double-Check 시스템을 거칩니다. (1) 시스템에 의한 GPS/시간 검증, (2) NLP 알고리즘에 의한 텍스트 구체성 분석을 통해 허수 데이터를 80% 이상 필터링합니다.

RTR Decay Logic

Time-Series
RTR_final = Sentiment_avg × e^(-λt)
λ(감쇠상수) = 0.035 (20일 경과 시 점수 50% 하락 기준)
t = 마지막 방문 후 경과일

// Normalization Implementation (Z-Score)
function normalize(score, mean, std) {
    const z = (score - mean) / (std || 1);
    // T-Score conversion for 0-100 scale
    return Math.min(100, Math.max(0, (z * 10) + 70));
}

C Appendix C. 성과 가중치(Outcome Weight) 및 상관관계 분석

왜 처방량(Quantity) 중심인가?

제약/바이오 시장에서 매출액(Revenue)은 '약가 인하'나 '대량 발주' 등 외부 변수에 의해 왜곡되기 쉽습니다. 순수 처방량(Quantity)은 현장에서의 마켓 쉐어(M/S) 장악력을 보여주는 가장 정직한 지표입니다.

"FGR(0.6*Qty + 0.4*Rev) 설계는 시장 상황이 악화되어도 실질적인 점유율을 지켜내는 '진짜 실력'을 선별하기 위함입니다."

E Appendix E. 데이터 기반 지능형 코칭 시나리오 및 처방(Prescription)

Scenario: The Ghost Hunter

High Activity, but Zero Quality/Outcome

Detection Logic: 방문 건수(Visit)는 상위 10%이나, HIR(품질) 점수가 하위 20%이며 고객 피드백(Question 키워드)이 전무할 때 발생.
Coaching Action: 무분별한 방문 중단 명령. 핵심 3개 계정에 대한 'Deep Dive' 면담 계획 수립 및 관리자 동행(Joint Visit) 요청.

Scenario: High Quality, Low Reach

Perfect Content, but Insufficient Volume

Detection Logic: HIR(품질) 및 RTR(관계) 점수는 최상위이나, BCR(빈도) 점수가 목표 대비 60% 미만으로 활동 생산성이 저조할 때 발생.
Coaching Action: 효율적 동선 계획(Route Optimization) 코칭. 성공 사례를 팀 내 템플릿화하여 '품질'을 '속도'로 전환하는 역량 전파.

The Goal of Data Coaching

"데이터는 사람을 평가하기 위한 도구가 아니라, 성공의 재현성을 높이고 리스크를 예방하기 위한 나침반이어야 합니다."
